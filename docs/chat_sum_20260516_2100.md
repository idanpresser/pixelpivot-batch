## 2. Comprehensive Changes

### A. TDD Feature Completion
We completed the pending task list strictly following Red-Green-Refactor cycles:
*   **Heuristic Calibration:** Populated `heuristic_table.json` with robust AVIF quality baseline data across all resolution buckets (small to xlarge) and image categories.
*   **Decoupled Analytics:** Separated heavy analytics imports (`pandas`) from the core `BatchRepository` into `analytics_api.py`. This significantly accelerated module load times and reduced the memory footprint of the API layer.
*   **Frontend Integration:** Wired the Streamlit GUI (`run_panel.py`) directly to the FastAPI backend using asynchronous `httpx` polling, complete with visual spinners for real-time batch tracking.
*   **Hot Folder Automation:** Built REST endpoints (`/hotfolder/register`, `/list`) and integrated a `watchdog` daemon singleton into the FastAPI startup lifecycle to automatically process incoming files.

### B. Architectural Refactoring & DB Portability
*   **Dual-Engine Abstraction:** Implemented a dialect-aware abstraction layer (`app/core/db/dialect.py`) that transparently handles parameterized queries for both **PostgreSQL** (`%s`) and **SQLite** (`?`).
*   **Schema Optimizations:** Updated `schema.py` to apply SQLite-specific pragmas (WAL mode, synchronous mode) dynamically.
*   **Heuristic Generator Tool:** Created `tools/generate_heuristic_data.py` to ingest historical benchmark data from any supported database, handling edge cases like type-casting JXL float distances vs. standard integer qualities.

### C. High-Performance Converter Stack
*   **Persistent Sharp Daemon:** Eliminated the massive overhead of spawning a Node.js process per image by creating `sharp_daemon.js`. We integrated this into Python via a persistent TCP socket.
*   **Socket Pipelining:** Upgraded the `SharpConverter.convert_batch` method to push hundreds of JSON conversion requests through the socket concurrently, drastically improving throughput.
*   **Circuit Breaker Pattern:** Engineered a safeguard into `BaseConverter`. If a specific tool (e.g., Magick) fails three times consecutively or encounters a fatal hardware error (OOM, CUDA crash), it is marked as `BROKEN`. The orchestrator now respects this state, preventing cascading system locks.

### D. Premium GUI Overhaul
*   **Legacy Aesthetic Migration:** Extracted and adapted the "Cyberpunk" design system from the legacy PixelPivot project (Void Black, BTC Orange, Cyan).
*   **Modular UX:** Refactored `main.py` into a clean, tabbed interface (EXECUTE, HOT FOLDERS, HISTORY).
*   **Custom Styling:** Injected custom CSS for gradient result cards, hover transformations, and an animated "Pulse" effect for active system states.

### E. Infrastructure, WSL, and Air-Gap Readiness
*   **Multi-Stage Dockerfile:** Built a highly optimized image bundling both Python 3.12 and Node.js 20, pre-loaded with `libvips`, `libheif`, and `ffmpeg`.
*   **WSL Scripts:** Created `scripts/wsl_start.sh` for one-touch stack deployment tailored for `/mnt/i/...` path mounting in Windows Subsystem for Linux.
*   **Air-Gap Export Utility:** Authored `scripts/export_airgap.sh`, which bundles the API, GUI, CLI, and Database images into `.tar.gz` files for offline transfer.
*   **Dedicated CLI Service:** Added `pixelpivot-cli` to `docker-compose.yml`, providing an interactive, persistent terminal environment loaded with all image processing binaries.
*   **UV Environment:** Transitioned the local python environment to `uv` for lightning-fast dependency resolution and test execution.

## 3. Pitfalls Avoided & Issues Handled

1.  **WSL Line Ending Conflicts ("Bad Interpreter"):**
    *   *Issue:* Scripts created in the Windows IDE contained `\r\n` (CRLF) line endings, causing Linux/WSL bash to fail with obscure `$'\r': command not found` errors.
    *   *Resolution:* Handled dynamically using `tr -d '\r'` via WSL commands to sanitize the shell scripts before execution, ensuring stable cross-platform script behavior.
2.  **Air-Gapped Font Rendering:**
    *   *Issue:* The premium GUI relied on Google Fonts (Inter, Space Grotesk, JetBrains Mono) loaded via CDN, which would break in an offline deployment.
    *   *Resolution:* Downloaded the TTF files locally, wrote a Python utility in `theme_engine.py` to Base64 encode the binaries, and injected them directly into the CSS via `@font-face`.
3.  **Missing System Dependencies for Testing:**
    *   *Issue:* Running the new `test_real_assets_end_to_end.py` locally caused `OSError` and `ImportError` because `libvips-42.dll` or `magick` were missing on the host Windows machine.
    *   *Resolution:* Implemented defensive programming in `pytest`. The tests dynamically check for the presence of binaries (`shutil.which`) or attempt a dry-run import/load. If they fail, the specific test safely `skips` rather than crashing the entire test suite, allowing development to continue locally while ensuring the Docker CI catches the full run.
4.  **Zombie Node.js Processes:**
    *   *Issue:* The Sharp daemon process could be left orphaned if the Python orchestrator crashed.
    *   *Resolution:* Tied the Node.js process lifecycle to the `stdin` pipe of the parent. Added `process.stdin.on('end', () => process.exit(0));` in the JS daemon, ensuring it instantly terminates if the Python host dies.
